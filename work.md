# Техническое задание
## Telegram Mini App «Меню ресторана Цзяюань»

### 1. Цель проекта

Создать Telegram Mini App, позволяющее пользователю:

- открыть мини‑приложение из Telegram
- просматривать меню ресторана в удобной структуре
- добавлять блюда в корзину
- оформить заказ, указав адрес и контакт
- отправить заказ администратору (в Telegram)

Это MVP, поэтому функционал должен быть максимально простым, быстрым и надёжным.

---

### 2. Основные разделы Mini App

#### 2.1. Главный экран

**Цель:** быстрый доступ ко всем категориям меню.

**Элементы:**

- Логотип/название ресторана
- Список категорий (карточки или плитки)
- Каждая категория содержит:
  - название (RU + CN)
  - иконку (можно использовать первую картинку из категории или дефолтную)

При клике → переход на экран категории.

#### 2.2. Экран категории

**Цель:** показать блюда выбранной категории.

**Элементы:**

- Название категории
- Список блюд (карточки):
  - фото блюда (image)
  - название (RU + CN)
  - вес
  - цена
  - кнопка «Добавить в корзину»

При клике на блюдо → экран блюда (опционально, можно пропустить в MVP).

#### 2.3. Корзина

**Цель:** показать выбранные блюда и оформить заказ.

**Элементы:**

- Список добавленных блюд:
  - название
  - цена
  - количество (с кнопками + / –)
- Итоговая сумма
- Поля ввода:
  - адрес доставки
  - комментарий (необязательно)
  - имя
  - телефон (можно автозаполнить из Telegram)
- Кнопка «Отправить заказ»

#### 2.4. Экран подтверждения

После отправки заказа:

- сообщение «Ваш заказ отправлен»
- кнопка «Вернуться в меню»

---

### 3. Логика работы

#### 3.1. Хранение данных

Меню загружается из статического JSON:

- **Файл:** `menu.json` — структура уже готова.
- Картинки хранятся в `/images/<code>.jpg`.

#### 3.2. Корзина

Корзина хранится локально в Mini App (в браузере Telegram):

- через **localStorage**
- или через state‑менеджер (например Zustand/Redux)

Сервер не обязателен для MVP.

#### 3.3. Отправка заказа

После заполнения формы Mini App отправляет заказ через:

**Вариант A (рекомендую)** — Telegram Bot API → `sendMessage`

Сообщение отправляется в личку администратору или в закрытый канал/группу.

**Архитектура бота:** Telegram-бот вынесен в отдельный сервис `services/telegram-bot` (NestJS + TypeScript, Docker). Работает в режиме webhook. При деплое на Vercel — бот не используется; webhook должен указывать на URL развёрнутого бота (Docker + Caddy).

Формат сообщения:

```
Новый заказ:
Имя: ...
Телефон: ...
Адрес: ...
Комментарий: ...

Блюда:
- G1 Жареная говядина с тмином × 2 = 44р
- F14 Чаджамьен × 1 = 16р

Итого: 60р
```

**Вариант B** — отправка на backend (если появится).

### 4. UI/UX требования

#### 4.1. Общие требования

- Минималистичный дизайн
- Быстрая навигация
- Большие карточки блюд
- Фото обязательны
- Цветовая схема: белый + красный/оранжевый (китайская кухня)

#### 4.2. Кнопки

- «Добавить в корзину» — яркая, заметная
- В корзине — кнопки +/– для изменения количества

#### 4.3. Адаптивность

Mini App должен корректно работать:

- на iOS
- на Android
- в десктопном Telegram

---

### 5. Технические требования

#### 5.1. Технологии

**Рекомендуемые:**

- **Frontend:** React + Telegram WebApp SDK
- **Стили:** TailwindCSS или обычный CSS
- **Сборка:** Vite
- **Хостинг:** у пользователя есть сервер с Node.js и NPM

Backend для MVP не обязателен.

#### 5.2. Интеграция с Telegram

Использовать:

- `Telegram.WebApp.initDataUnsafe` — для получения данных пользователя
- `Telegram.WebApp.MainButton` — для кнопки «Корзина»
- `Telegram.WebApp.sendData()` — для отправки заказа боту
